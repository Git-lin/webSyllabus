基本语法

JavaScript和很多计算机语言都有一个相似之处，就是执行代码的单位为行，和HTML文档流类似，也是一种从上到下一行行地执行的语言，通常一行代码完成一个功能。但这一行行的代码很多时候是存在不同的作用域，而且会涉及到代码的封装/调用等。但无论什么样结构的JS（JavaScript的简称，后续内容中的“JS”同此），从执行单位区域来看，仍然是按顺序一行行执行的，这为我们阅读代码提供了很大的便利。

JavaScript代码执行的单位为行，而每一行通常是一条语句组成的，每一条语句的结束和CSS一样，都是由英文分号“;”标注的，不同的语句串联起了JavaScript强大而灵活的功能。

这些都可以叫做JavaScript的语句：

  • 1 + 3;    // 4
  • "apple";    // "apple"
  • 2<3;    // true
  • a = b + c;    // “b + c”的结果,“b”和“c”都必须为一个数字，否则返回“NaN”
  • d = 5 * f;    // “5 * f”的结果，“f”都必须为一个数字，否则返回“NaN”
  • e == 6;    // 如果“e”为6结果为“true”，否则为“false”
  • ;    // 空语句

有些语句单独出现并没有什么意义（如上列表中的“1+3”，“apple”等），它必须要在特定环境内使用才能产生对程序的实际意义，否则它们只会成为你程序中的“僵尸代码”。

对于一个初学者来说，什么时候该用什么样的语句，将语句放置于什么地方，怎么去组织自己的各条语句都是一个难点。要克服这些难点，首先就是要对JavaScript中各种基本语句有一个主观而正确的认识。只有认识并了解这些基本语句的作用，才能真正地将他们运用自如，只要稍微接触过计算机程序一段时间的人都应该有这样的认识。

JavaScript中的注释

JavaScript和HTML不同，到HTML5后有大量的语义化标签，再加上HTML文件的内容，只需要很少量的注释就能搞清每一块代码的作用。和CSS也不同，现在的浏览器都有不错的“开发者工具”，只要通过该工具就能找到浏览器显示内容所对应的CSS内容和相应的行号，需要的注释也只是区域性的注释。JavaScript很难通过现有的手段找到对应的JS代码，而且JS代码内部本身就涉及到大量的封装、调用，特别是在一些现在JS的MVC框架中，找到浏览器显示内容对应的JS代码真是一个“大工程”。所以代码的注释对于JavaScript这种编程语言来说显得就十分重要了。有位业内著名的开发工程师说过一句类似这样话，“代码中的注释量至少占代码量的三分之二”。虽然听起来是有些夸张，但只要有过一段编程经验的开发者都知道，如果一个项目中的程序部分没有注释，或者注释很少，那整个项目的维护难度是相当高的，所花的开发维护成本、沟通成本都是非常高的，甚至远远超过编写代码本身。本想节约的时间反而变成了后面测试、维护、修改bug的巨大时间成本，造成的浪费和亏损都非常大。

针对这一点，JavaScript本身在语言设计上也和其它编程语言一样没有含糊，不像HTML这种标签语言或者CSS这种样式设计文件一样只有一种注释，JavaScript考虑到多种应用场景定义了两种注释方式：

单行注释
这种注释只需要在代码或文本前加上双斜杠“//”即可，不需要结束注释符，比较方便。这种注释通常是用于说明某条语句或者某个变量的作用，亦或者是说明参数或属性中可配置的值，可能返回的结果等。
多行注释
多行注释的写法和CSS内的注释写法完全一样，采用以“/*”开始，以“*/”结束的注释方式。通常的用法是暂时让某一段代码不再生效，或者是在开发阶段做区域单元测试的时候用得比较多。现在的团队开发中越来越重视代码的规范性，和代码的功能性说明，所以，多行注释现在也比较多地用来说明单个JavaScript的主要功能和用来做区域划分。

常量和变量

定义一个变量是JS里基本语法之一，变量的存在使得语句和数据变得可操控。之前我们提到过，一个语言单独存在是没有意义的，很多时候要使这些语句变得有意义，能被程序使用，参与到整个逻辑结构中去，那就得使用一个叫做“变量”的东西。在程序中有两种“量”，一种被称为“常量”，一种被称为“变量”。

常量
在程序中不会被更改的数据，如数值“1,2,3...”和字符串就是编程术语中最常用的“常量”，通常上它们在程序中并没有单独存在的意义，只有在特定的表达式才能发挥它们的作用。
变量
相比常量，变量在JS中扮演着更加重要的作用，这种重要并不是一种可有可无的关系，而是一种主导性强弱的关系。“常量”是组成“变量”最基本的单位，而“变量”又由“常量”、“常量运算”、“常量+变量”、“常量运算+变量”或“变量+变量”的表达式组成。变量在JS中，从被定义到使用通常是会发生值的变化的，而这种变化有的时候不止一次，会根据程序的逻辑做出不同的变化，正是因为这种变化的特性，“变量”因此得名。每一个变量都必须有一个“变量名”。

变量的定义，实质上是对一个“值的引用”。如上例中的“var a = 5;”，实质上是将变量“a”和常量“5”建立起一种引用关系。在编程中我们把这种引用关系叫做“赋值”。而“var”在这里的作用是声明创建一个变量，在当前的运行环境中被存储起来。

对于变量通俗的理解，如果变量相当于一个“鼠标”，那常量就相当于鼠标内的各种“元件”，这些元件的拼装成为了一个拥有特定功能的鼠标，而用不同的元件拼装，鼠标的性能和外观也可能会不一样。而用鼠标的作用是点击（或滚动）操作系统中的内容，但至于什么时候点击，点击什么内容，这个由用户决定。这相当于变量的作用由程序中的逻辑决定，决定什么时候使用，使用在什么位置一样。

JavaScript中的六种数据类型

数据类型决定了如何将代表这些值的位存储到计算机的内存中。和其它编程语言（如：C语言、Java等）一样，JavaScript也有自己的数据类型，代表的作用上和其它编程语言大致相同，但也有一些自己的特点。JavaScript总共有6种数据类型：

number（数值型）
该类型是用于表示阿拉伯数字的数据类型，可以是“0”、正数、负数、浮点数（小数）。数值型的值可以参与数学运算。
string（字符串）
该类型用于表示如“英文”，“中文”，“其它国家语言”，“键盘上的符号”，“特殊编码”等字符，甚至可以是数字。该类型的值都需要用一对英文引号（可以是单引号，也可以是双引号）阔起来，否则在使用该变量的时候会报出未定义的错误，成为一个“未定义”类型，而本想作为字符串的数字也会成为一个“数值型”。
boolean（布尔型）
该类型只有两个值，即：
  • true：表示“是”，在判断语句中可以用“非0”、“非undefined”、“非null”、“非NaN”、“字符串”、“对象”甚至是一个“函数”表示
  • false：表示“否”，在判断语句中可以用数值“0”、“undefined”、“null”或“NaN”表示
object（对象）
该类型用于表示一个对象，在JavaScript中并没有“数组”这个类型，所以数组在JavaScript里也作为一个对象，空值“null”也是一个对象。在JavaScript中可以通过语句“new Object()”来创建一个对象来赋给某一个变量。以下为对象的示例代码：

function（函数）
该类型表示变量的类型为一个可运行的“函数”。一个函数内包含一组语句（至少一条语句），通过这些语句的组合去实现一个特定的功能。在JavaScript中函数主要有以下分类：
  • 普通函数
  • 构造函数
  • 变量函数
  • 匿名函数
  • 闭包

变量的命名

JS的命名和我们之前所接触到的CSS有所不同，它的命名应当遵守以下规则：

  • 只能以“英文字母”、“其他国家语言字符”、“_”（下划线）、“$”（美元符号）开头
  • 不能以“数字”及“其它特殊符号”（键盘上的符号）作为开头
  • 首个字符以后可以使用“数字”
  • 不能使用“关键字”和“保留字”作为变量名
  • 不是保留字，但具有一定意义的字符，如“Infinity”、“null”、“NaN”、“undefined”等

和之前我们提到过的CSS的“id”、“class”和“name”的命名一样，JavaScript的命名也应该尽量“语义化”一点，即所命的名应该能尽可能真实地反应出该变量在程序中的实际作用。举几个实际的命名例子供参考：

  • 导航按钮组：navBtnGroup（Navigation button group）
  • 当前窗口的高度：win_h、winH、winHeight（window height）
  • 某个算术表达式运算的结果：result、count
  • 爱好：userHobby
  • 登录框/注册框：loginBox/regsterBox
  • 登录框状态：loginStatus
  • 用户名：userName
  • 用户的信息：userInfo（user information）
  • 信息提示框：mesgBox（message box）

当然也可以借鉴一些语言的命名方式，即已知变量的结果是一个什么数据类型的时候，在变量名前再加上一个类型标识符，这样能让开发者在维护和测试代码的时候更加清楚变量的一个实际作用，也能为思维构建提供一定的帮助。实际上在很多对代码规范要求很严格的大型企业都会有类似的命名规定。上面列表中的命名方式采用这种“数据类型”+“作用说明”的写法后，应该改为：

  • oNavBtnGroup（对象，object）
  • numWinHight（数值，number）
  • numResult（数值，number）
  • arrUserHobby（数组，object，在其它编程语言里是表示“array”）
  • oLoginBox/oRegsterBox（对象，object）
  • boolLoginStatus（布尔值，boolean）
  • strUserName（字符串，string）
  • oUserInfo（对象，object）
  • funMesgBox（函数，object）

函数内的命名很多时候可以稍微简单一些，只要函数本身有比较详细的注释，变量名简短一点有的时候反而更能帮助代码的阅读，但也需要掌握一个平衡点，不能无顾忌地缩减变量名。

当然，算术运算符和字符串拼接运算符一样，它也是可以在一个语句内出现多次，形成一个复合算术运算符，这种运算符通过小括号“()”去控制优先级。需要特别指出的是JavaScript内是没有数学中的“中括号”和“大括号”的算术运算符的，只能通过多层小括号去实现，括号必须成对出现，否则程序会出现异常。如“x = ((2 + y) * 5 - 10) / 2”。
赋值运算符
赋值运算符是为JavaScript中的变量赋值，即将该运算符右方计算出的值赋予左侧的变量。在JavaScript中有以下赋值运算符：（以下所有运算符中的变量“x = 2, y = 4”）
运算符 作用 实例 相当于 结果
= 赋值 x = y x = y x = 4
+= 变量与值相加 x += y x = x + y x = 6
-= 变量与值相减 x -= y x = x - y x = -2
*= 变量与值相乘 x *= y x = x * y x = 8
/= 变量与值相除 x /= y x = x / y x = 0.5
%= 变量对值取余 x %= y x = x % y x = 2

在很多稍微复杂一点的表达式中，以上几种运算符基本上是同时存在于一条语句中的。首先要熟悉这几种运算操作符，然后在项目中灵活地组合其用法，将其组合为性能最好，可扩展性最好的程序。

二元运算符

二元运算符（也称作“逻辑运算符”）是JavaScript用于判断几条语句成立情况的一种运算符，这种运算符之所以被称作“二元”，是因为这种运算符要成立，必须要具有两个“量”（除了“非”运算，这个“量”可以常量或变量），这种运算符会根据当前操作数返回的布尔值类型来决定返回的结果。常用于条件判断语句和函数参数的默认值配置。

二元运算符有三个，即“逻辑与”、“逻辑或”、“逻辑非”，有时为了简便会称作:与“&”、或“|”、非“！”（除了这里的“非”，实际上它们与加上了“逻辑”二字的表示的意思是有区别的，它称作“位运算符”，是对二进制数据进行操作的运算符）。

逻辑与——“&&”
只有当参数运算的所有变量或常量返回的值都为“true”的时候（每种数据类型返回的布尔值可以参看本章中“JavaScript的六种数据类型”章节中“boolean（布尔型）”的介绍），该运算符返回的最终结果才为“true”，否则为“false”。应用场景：“登录操作”，“完胜判定”，“七龙珠模式”。
逻辑或——“||”
参数运算的变量或常量中，只需要其中一个的值为“true”，该运算符返回的最终结果就为“ture”，如果所有值都为“false”的时候，那该运算符返回的结果才会为“false”。应用场景：“全勤绩效考核”，“棋牌游戏”。
逻辑非——“!”
如果当前常量或变量不是一个布尔值，将其通过布尔型转换的规则转换为一个布尔值（如字符串不是一个布尔型，但是经过规则转换为就会成为一个“true”，又如“undefined”经转换后会成为“false”），然后对当前的布尔值进行“取反”操作。如‘与--“&&”’代码中的示例。
