获取和设置属性

基本语法：

// 获取元素的属性 $("form input").attr("type"); // 设置元素的属性 $("form input").attr("type", "password");

同样是对属性的操作，和原生JavaScript的getAttribute和setAttribute不用的是这个方法是一个获取和设置“二合一”的方法，jQuery通过识别参数个数来执行不同的操作，当参数只有一个的时候表示获取属性，当参数个数为两个的时候表示设置属性，这和jQuery的css方法用来获取或设置标签的行内style样式属性是一样的原理。

获取和设置状态属性

状态属性和之前的普通属性都是HTML标签上的属性，区别在于状态属性在HTML5规范里是没有属性值的，而在严格模式的XHTML中它的属性值和它自身相等。这类属性在表单元素内比较常见，如：checked、disabled、readonly等，另外还有HTML5规范中新增的hidden、data-*，这些属性都是可以不用设置属性值的。而HTML5规范中新增的draggable、contenteditable和spellcheck这三个需要设置true和false值的属性，他们同样也属于是状态属性。

基本语法：

// 获取元素的状态属性 $("form input:checkbox").prop("checked"); $("form input:text").prop("readonly"); // 设置元素的状态属性 $("form input:checkbox").prop("checked", true); $("form button").prop("disabled", false);

通过attr()方法也能设置状态属性，但是却不能切换状态（true和false之间切换），也就是说它可以让一个多选框被选中，却不能让它取消选中，所以要去做类似的状态切换操作需要用到prop()这个方法。

移除属性和状态属性

		
基本语法：

// 移除元素的属性
$("div").removeAttr("style");

// 移除元素的状态属性
$("form input:checkbox").removeProp("disabled");

    	
有的时候通过jQuery操作过的Element标签会残留一些属性，如通过操作class和操作css的方法，当标签上的属性值已经为空的时候，class属性和style属性仍然存在，这可能会影响后续的一些判断操作，这个时候就可以用这两个方法来彻底删除这些属性。当然，对于已经拥有属性值的属性来讲，这两个方法也是能够正确生效的，这样会移除这些属性值的设置效果。

    

    
    
    

    	
添加和移除Class

    	
jQuery操作class的方法和ES5出现了classList对象里的方法的功能很相似（如：add、remove、toggle等），但却更加简单直观。通过这个方法可以配合已经在CSS样式表里设置好的样式来简单完成很多本来应该是比较繁复的界面交互效果。

		
基本语法：

// 为元素添加Class
$("nav ul a").addClass("active");

// 移除元素的Class
$("nav ul a").removeClass("active");

		
无论是在原生的JavaScript中还是在jQuery中，有的时候为元素对象添加或移除class并不意味着一定是实现样式的切换操作，而也有可能是实现状态的切换。通过切换或者设置状态，可以让一些不相关的组件实现间接的联系（这在“章节练习二”中有比较明显的体现）。

		
对于Class的操作，jQuery还提供了一个非常方便的方法，该方法的作用是，如果该元素含有参数内的Class则移除，否则添加参数内的Class，这样一来就可以省略掉一些条件判断，让代码的可读性和可维护性都更高。它基本语法如下：

// 如果含有active这个Class则移除，否则添加这个Class
$("nav ul a").toggleClass("active");
